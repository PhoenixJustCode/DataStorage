# Лабораторная работа 9: MongoDB

## Цель
Изучить основы работы с документоориентированной СУБД MongoDB: CRUD-операции, операторы запросов, агрегации, принципы проектирования схем и индексирование.

## Установка и запуск


## Что реализовано

### Part 2 — Sample Dataset
Две коллекции: `products` (5 документов) и `orders` (4 документа). Заполняются автоматически при запуске.

### Part 3 — CRUD
- **Find**: все продукты; проекция (name, price); фильтр + сортировка
- **Insert**: добавление Bluetooth Speaker
- **Update**: `$mul` — увеличение цены на 10%; `$push` — добавление тега в массив
- **Delete**: удаление по `_id`; удаление по условию (`stock < 10`)

### Part 4 — Operators
- `$gte/$lte` — диапазон цен
- `$in` — множественные статусы
- Поиск по элементу массива (`tags`)
- `$ne` — исключение значения

### Part 5 — Aggregation Pipeline
1. Общая выручка (`$group` + `$sum`)
2. Средний чек по клиенту (`$avg`)
3. Топ продуктов — `$unwind` → `$group` → `$lookup` (join с products)
4. Заказы по месяцам (`$year`, `$month`)

### Part 6 — Embedding vs Referencing (теория)

**Сценарий A (блог + комментарии):**
Лучше **отдельная коллекция** (referencing). Комментарии нужно агрегировать отдельно (количество на пользователя), документ поста может расти неограниченно. Ссылка через `postId` в каждом комментарии.

**Сценарий B (товар + отзывы):**
Однозначно **referencing**. Сотни отзывов раздуют документ (лимит 16 МБ), пользователи ищут свои отзывы отдельно. Коллекция `reviews` с полем `productId`.

### Part 7 — Indexing (теория)

1. **Индекс на `customer` в `orders`** — без индекса MongoDB сканирует все документы (collection scan). С индексом сразу переходит к нужным записям, что сокращает время с O(n) до O(log n).

2. **Рекомендуемые индексы для `products`:**
   - `category` — частые фильтры по категории
   - `price` — сортировки и диапазонные запросы ($gte/$lte)
